# Project Metadata

## Project Name
**NI AI Assistant**

## Versioning Strategy
- **Strategy:** Semantic Versioning (`MAJOR.MINOR.PATCH`).
- **MAJOR** increments for breaking changes (architecture changes, incompatible prompt contracts, or user workflow changes).
- **MINOR** increments for backward-compatible feature additions (new commands, new prompts, added runbook sections).
- **PATCH** increments for backward-compatible fixes (clarifications, typo fixes, non-breaking prompt tuning).
- **Tagging convention:** `vX.Y.Z`.

## Owners
- **Product Owner:** Owns priorities, scope, and handoff criteria.
- **Technical Owner:** Owns implementation quality, architecture consistency, and release readiness.
- **Prompt/AI Owner:** Owns system prompt safety, behavior quality, and prompt update validation.
- **Operations Owner:** Owns deployment process, rollback readiness, and operational documentation.

> Replace role placeholders with named individuals/teams as your governance model matures.
